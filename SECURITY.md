# Security

## Supported versions

Security fixes are expected to be applied to the `main` branch. Self-hosters should track upstream and redeploy when updates are published.

## Reporting a vulnerability

Please **do not** file a public issue with vulnerability details, exploit steps, or live accounts.

If this repository has a security policy or private reporting channel in GitHub, use that. Otherwise, contact the maintainers through a private channel. Include enough information to reproduce and assess impact.

## Scope and expectations

This is application-level security for a self-hostable file-sharing stack. Hardening (TLS, WAF, secrets management, DDoS, rate limits, backup policy) is the operator’s responsibility in production.
